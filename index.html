Problems With Image Sprites and Icon Fonts

  • Both have sizing issues in certain browsers and edge cases

  • Both are troublesome to position correctly

  • Icon fonts anti-alias differently between browsers

What are SVG Sprites?

  • An SVG sprite is a single SVG file that contains data for multiple SVG graphics.

  • SVG sprites are convenient because they:
    • Serve with one HTTP request
    • Are simple to position
    • Allow you to style them with CSS
    • Easily added to your workflow
    • Are easy create fallbacks for

What does an SVG Sprite look like?

<!DOCTYPE html>
<html>
<head><title>My Awesome Page</title></head>

  <svg version="1.1" xmlns="http://www.w3.org/2000/svg" style="display:none;">
    <defs>
      <symbol id="icon-one" viewBox="0 0 80 32">
        <path class="path1" d="M0 10h32l-2 20h-28l-2-20zM29 6l1"></path>
      </symbol>

     <symbol id="icon-two" viewBox="0 0 80 32">
       <path class="path1" d="M26 30l6-16h-26l-6 16zM4 12l-4 18v-26h9l4"></path>
     </symbol>
   </defs>
  </svg>
<h1>Sprites, fool.</h1>

  <svg>
    <use xlink:href="#icon-one"></use>
  </svg>

<body>
</html>

<svg><use>

  • <use> tag calls width, height and viewbox settings from the symbol tag in sprite.
  • Elements that render inside of <use> tag are treated as Shadow DOM
    • Fun fact: SVG used the RCC specification to first implement Shadow DOM and Shadow DOM functionality over a decade ago.
  • Requires fallback to work in IE. ¯\_(ツ)_/¯

<defs> Tag

SVG Sprite Inception

  • The <defs> tag can be used to define an element without rendering it

  • Can store anything (e.g. patterns, gradients, masks, etc...)

  • Great for creating minimal templates

<defs> Tag

SVG Sprite Inception

<svg>
  <defs>
    <linearGradient id="gradient">
      <stop offset="0%" style="stop-color: white"></stop>
      <stop offset="100%" style="stop-color: black"></stop>
    </linearGradient>
  </defs>

  <circle cx="50" r="40" stroke="black" stroke-width="1" fill="url(#gradient)" />
</svg>
